## 构建模型

```


基本原则：
1. 明确目的
2. 明确输入输出
3. 寻找材料-完整描述-更多例子
4. 验证
5. 修正

概念：
概念类别：
名词  --  分类
动词  --  过程
不同的概念内部会有不同的结构，不同的知识也会有不同的结构。但是他们最底层的本质是一样的，概念是变量，而具体的事物是常量。知识是 2 类事物或以上之间的通用变化规律，例子是具体事物之间的规律。

例子-1
知识名词化，比如，刚刚看了 Set 如何保证数据不重复。这个肯定是一个处理过程，但是这个过程会用一个名词代替，然后你理解了过程，最后只需要记住这个名词就够了，通过这个名词回忆起处理的过程。
然后，这里的 Set 是一个接口，也就是抽象的概念，下面还有子抽象概念，每个子概念的处理方式也不一样。
输入是：子 Set 类
关系是：具体的处理方法
输出是：是否是唯一的数据
把抽象的概念，分解，降低层级，变成具体的对象。

例子-2：
别人可以学会，只有你弄明白了别人会什么可以学会，你才能学会。比如面试，为什么别人可以，找到背后的通用原则，然后复制就可以了。
比如，社交也是，肯定有通用的规则，但是如果没用人教你，你只能去观察，去看优秀的人是怎么社交的。你不能凭空看理论就学会。如果没有见过优秀的人，那你没用学习的经验可以模仿。凭空思考是学不会的，需要实际的场景。

概念分析
事务：这个是结果，不确定的状态，也是一个概念。
编码：动词，处理问题的过程，把一个状态变成另一种状态。
判别模型：处理问题的过程。
联结模型：处理问题的过程。
变量：不确定的状态，概念。
常量：确定的状态，具体的对象。

JVM
内存：名词，不确定的状态，变量。
把数据存储到内存中。
堆：名词，变量，可以再分解，内部还有各种子结构。


分类视角
问题
材料的类型

什么是
为什么是
目的是
怎么得到


目的：
构建判别能力。
这里的【判别能力】是一个抽象的概念，可以细分出更具体的类别。但不管是判别什么，本质上是一种判别能力。

向上-向下判别

向上判别：
1. 判断一个具体的对象是否属于上一层的抽象概念。

向下判别：
1. 判别一个概念有哪些子类别。

感觉描述的不清晰，不准确。

不同的判别需要不同的方式构建



how：
根据【具体对象】是否具有【抽象概念】的内涵来判断是否属于该概念。
怎么确定概念的内涵？

至少需要两个对象，具体对象，抽象概念，还有概念的内涵（对象的共性）。

然后具体对象可能也是抽象的，或者是由不同概念组合而成的。所以，还需要继续向下具象。

什么时候，选择构建什么样的判别能力，

一定需要借助例子，实例来理解抽象材料，否则不可能真正理解。

举例：
1. 根据你的面试情况选择是否录用。
2. 银行根据你的信用选择贷款给你。
3. 根据你是否建立了索引来判断查询效率如何。
4. 根据你是否是中国人来判断你会不会普通话。
5. 根据是几边形来判断总角之和的度数。

这里好像不太对，不是分类对象，而是处理概念。

特点：
结构：主谓结构

知识的 root
目的是根，而过程是不需要记忆。

一阶
步骤：
1.关键字
2.是什么
3.作用是什么


二阶
步骤：
1.关键字
2.拆分关键字
- 组合：子知识
- 顺序：步骤
- 条件：什么情况下成立
- 循环：是否同一知识的多次应用
3.循环拆分子知识

比如：
索引，这是一个二阶知识，需要拆分成更子知识。
子知识的结构是不一样的。


process
1. 例子
2. 明确输入输出
3. 寻找共性
4. 明确问题：what-why-goal-how
5. 分而治之
6. 视角-属性
```
